**核心指令**：请仅针对现有代码库进行研究和文档记录，务必保持客观。**严禁**在当前阶段提供任何改进建议或评价。

## 行为契约 (Behavior Contract)

- **仅记录现状**：只记录今天实际存在的代码逻辑；不要建议改进方案、不要进行根因分析 (RCA)，也不要规划未来工作。
- **提供精确引用**：在所有发现中必须提供具体的文件路径和行号引用。
- **深度阅读优先**：在分解工作任务之前，必须完整阅读用户提及的所有相关文件。

## 执行步骤 (Research Steps)

1. **全量阅读**：完整阅读用户直接提及的文件。
2. **任务分解**：将查询分解为多个专注的研究领域，并创建一个内部核对清单。
3. **并行探索**：当研究领域相互独立时，并行探索相关目录和文件。如有必要，请查阅 `git` 历史记录以寻找问题的答案。
4. **集成第三方文档**：如果需要参考第三方 API 来理解工作原理，请务必使用 `context7` 工具获取最新的 API 文档。
5. **综合汇总**：在所有探索完成后进行综合分析；**实时代码的发现优先级高于历史文档**。
6. **产出文档**：生成一份结构化的研究文档，并将其保存在项目根目录的 `research/` 文件夹下。

## 研究文档内容要求

- **研究摘要**：回答核心问题的发现总结。
- **详细章节**：按组件或区域划分的详细分析，并附带代码引用。
- **关联分析**：说明跨组件（如 `mall-common` 与 `mall-admin`）的连接方式与数据流向。

## 输出格式规范 (Output Format)

- **高层摘要**：提供 3 到 6 句话的精炼总结。
- **详细结果**：按组件或区域分组呈现。
- **代码引用格式**：统一采用 `模块名/src/main/java/路径/文件名.java:起始行号-结束行号` 的形式。
